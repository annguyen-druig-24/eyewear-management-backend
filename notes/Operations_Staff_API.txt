### OrderDetail page for OPERATIONS STAFF ###
1) Các status của table Shipping_Info và table Order
Shipping_Status: PENDING, PACKING, SHIPPING, DELIVERED, FAILED, CANCELED, RETURNED
Order_Status: PENDING, CONFIRMED, PARTIALLY_PAID, PAID, PROCESSING, READY, COMPLETED, CANCELED

2) Mapping status - các nút bấm cập nhật trạng thái cho OPERATIONS STAFF: 	
	a) Order = CONFIRMED:
		- Nếu có prescription : hiện nút Bắt đầu gia công → set Order=PROCESSING .
		- Nếu không prescription : hiện nút Bắt đầu đóng gói → set Shipping=PACKING .
	b) Order = PROCESSING (chỉ nhóm có prescription):
		- Hiện nút Chuyển đóng gói → set Shipping=PACKING .
	c) Shipping = PACKING:
		- Hiện nút Bàn giao GHN → set Order=READY , Shipping=SHIPPING .
	d) Shipping = SHIPPING:
		- Không cho staff set tay DELIVERED nếu đã có webhook GHN; chỉ cho Đồng bộ GHN (manual sync) khi cần.
	e) Shipping = DELIVERED
		- Nếu đơn cọc ( Order=PARTIALLY_PAID ): hiện Xác nhận đã thu đủ → Order=PAID , sau đó Hoàn tất đơn → Order=COMPLETED .
		- Nếu đơn đã paid/full COD thu đủ: hiện Hoàn tất đơn → Order=COMPLETED .
	f) Order/Shipping = CANCELED, COMPLETED
		- Không hiện nút update nghiệp vụ (read-only).

3) Phân biệt các trạng thái do OPERATIONS STAFF, GHN cập nhật:
	a) Do OPERATIONS STAFF cập nhật:
		- Shipping: PENDING -> PACKING (bắt đầu xử lý nội bộ).
		- PACKING -> SHIPPING (khi bấm bàn giao và đã tạo vận đơn GHN thành công).
		- Với đơn prescription: Order: CONFIRMED -> PROCESSING trước bước PACKING.
	b) Do GHN tự động cập nhật (qua webhook/poll):
		- Shipping -> DELIVERED , FAILED , RETURNED (theo tracking thực tế).
		- Khi nhận DELIVERED , backend mới chạy rule nghiệp vụ tiếp: PAID (nếu còn thiếu) rồi COMPLETED .

				1. `START_PROCESSING` → `Order_Status: CONFIRMED → PROCESSING`
				2. `MOVE_TO_PACKING`  → `Shipping_Status: PENDING → PACKING`
				3. `HANDOVER_TO_GHN`  → `Order_Status: ... → READY`, `Shipping_Status: PACKING → SHIPPING`
				4. `MARK_DELIVERED`   → `Shipping_Status: SHIPPING → DELIVERED` (+ chốt payment/invoice, note của Backend)
				5. `COMPLETE_ORDER`   → `Order_Status → COMPLETED`
		* Sẽ hiển thị các nút bấm như thế nào?
			1) FE lấy danh sách nút cần hiển thị từ `result.availableActions` (API `GET /api/operation-staff/orders/{orderId}` hoặc từ response sau khi PUT status).
			2) Mapping theo trạng thái hiện tại:
				a) Nếu `Order_Status = CONFIRMED`:
				  - Nếu đơn có prescription → hiển thị `START_PROCESSING`
				  - Nếu đơn không có prescription và `Shipping_Status = PENDING` → hiển thị `START_PACKING`

				b) Nếu `Order_Status = PROCESSING` và có prescription và `Shipping_Status = PENDING`:
				  - hiển thị `MOVE_TO_PACKING`

				c) Nếu `Shipping_Status = PACKING`:
				  - hiển thị `HANDOVER_TO_GHN`

				d) Nếu `Shipping_Status = SHIPPING`:
				  - hiển thị `MARK_DELIVERED`, `MARK_FAILED`, `MARK_RETURNED`

				e) Nếu `Shipping_Status = DELIVERED`:
				  - hiển thị `COMPLETE_ORDER`

				f) Nếu đơn ở trạng thái read-only:
					- `Order_Status = CANCELED` hoặc `COMPLETED` hoặc `Shipping_Status = CANCELED` → `availableActions = []`
		* Các rule chi tiết của từng case:
			Nếu không thỏa điều kiện, API sẽ throw `INVALID_REQUEST`.
			### `START_PROCESSING`
			- Điều kiện:
			  - `Order_Status == CONFIRMED`
			  - `hasPrescriptionItem == true`
			- Kết quả:
			  - `Order_Status = PROCESSING`
			### `START_PACKING`
			- Điều kiện:
			  - `Order_Status == CONFIRMED`
			  - `hasPrescriptionItem == false`
			  - `Shipping_Status == PENDING`
			- Kết quả:
			  - `Shipping_Status = PACKING`
			### `MOVE_TO_PACKING`
			- Điều kiện:
			  - `Order_Status == PROCESSING`
			  - `hasPrescriptionItem == true`
			  - `Shipping_Status == PENDING`
			- Kết quả:
			  - `Shipping_Status = PACKING`
			### `HANDOVER_TO_GHN`
			- Điều kiện:
			  - `Shipping_Status == PACKING`
			- Kết quả:
			  - `Order_Status = READY`
			  - `Shipping_Status = SHIPPING`
			- Ghi chú:
			  - **Không gọi GHN create order**.
			### `MARK_DELIVERED`
			- Điều kiện:
			  - `Shipping_Status == SHIPPING`
			  - Nếu có `expectedDeliveryAt` thì chỉ cho phép khi **ngày hiện tại >= ngày của expectedDeliveryAt**
			- Kết quả:
			  - `Shipping_Status = DELIVERED`
			  - Chốt thanh toán:
				- Tất cả `Payment.status = PENDING` → `SUCCESS`
				- `Invoice.status: UNPAID/PARTIALLY_PAID` → `PAID`
			### `MARK_FAILED`
			- Điều kiện:
			  - `Shipping_Status == SHIPPING`
			- Kết quả:
			  - `Shipping_Status = FAILED`
			### `MARK_RETURNED`
			- Điều kiện:
			  - `Shipping_Status == SHIPPING` hoặc `DELIVERED`
			- Kết quả:
			  - `Shipping_Status = RETURNED`
			### `COMPLETE_ORDER`
			- Điều kiện:
			  - `Shipping_Status == DELIVERED`
			  - Nếu có `expectedDeliveryAt` thì chỉ cho phép khi **ngày hiện tại >= ngày của expectedDeliveryAt**
			- Kết quả:
			  - `Order_Status = COMPLETED`
			  - Chốt thanh toán (chạy lại để đảm bảo không lệch):
				- Tất cả `Payment.status = PENDING` → `SUCCESS`
				- `Invoice.status: UNPAID/PARTIALLY_PAID` → `PAID`
