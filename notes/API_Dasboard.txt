API DASHBOARD - FLOW + TEST CASES (CHI TIET, DE HIEU)
====================================================

1) Tong quan
- Base path: /api/v1/dashboard
- Co 2 endpoint chinh:
	+ GET /api/v1/dashboard
	+ GET /api/v1/dashboard/top-products
- Date query format: yyyy-MM-dd
- Neu khong truyen ngay: service tu set khoang mac dinh 7 ngay gan nhat


2) Security va quyen truy cap

2.1 GET /api/v1/dashboard
- Controller co @PreAuthorize("hasAnyAuthority('ROLE_SALES STAFF','ROLE_ADMIN','ROLE_MANAGER')")
- Nghia la endpoint nay can JWT hop le + role dung
- User role khac (vd CUSTOMER) -> 403
- Khong token -> 401

2.2 GET /api/v1/dashboard/top-products
- Endpoint nay duoc expose public trong SecurityConfig:
	+ /api/v1/dashboard/top-products nam trong PUBLIC_GET_ENDPOINTS
- Khong can token van goi duoc


3) Luong xu ly nghiep vu trong service

3.1 Rule validate ngay (ca 2 endpoint)
- Neu startDate va endDate cung co, ma startDate > endDate:
	+ throw IllegalArgumentException
	+ message:
		* getDashboardStatistics: "Ngày bắt đầu không được lớn hơn ngày kết thúc!"
		* getTopSellingProducts: "Ngày bắt đầu không được lớn hơn ngày kết thúc!"
		* getTopSellingProductsByType: "Start Date must before End Date!"

3.2 Rule chuan hoa khoang thoi gian
- endDateTime:
	+ co endDate -> endDate at 23:59:59.999999999
	+ khong co endDate -> now()
- startDateTime:
	+ co startDate -> startDate at 00:00:00
	+ khong co startDate -> endDateTime minus 6 days, set 00:00:00
- Tuc la khoang mac dinh la 7 ngay (tu ngay -6 den ngay hien tai)

3.3 Luong GET /api/v1/dashboard (Dashboard summary)
- Tinh summary:
	+ totalRevenue = SUM(totalAmount) cua order status IN (COMPLETED, PAID)
	+ pendingOrders = dem orderStatus=PENDING trong khoang ngay
	+ completedOrders = dem orderStatus=COMPLETED trong khoang ngay
- Tinh revenueChart:
	+ Lay raw query theo ngay (dd/MM) chi cho COMPLETED, PAID
	+ Truyen vao Map<label, revenue>
	+ Loop tat ca ngay tu start -> end
	+ Ngay nao khong co du lieu => chen revenue = 0
	+ Dam bao chart khong bi khuyet ngay
- Tinh orderStatusChart:
	+ Dem so order theo tung status trong khoang ngay
	+ Mapper translate status sang tieng Viet:
		* PENDING -> Đang xử lý
		* COMPLETED -> Hoàn thành
		* CANCELED -> Đã hủy
		* PAID -> Đã thanh toán
		* PARTIALLY_PAID -> Thanh toán một phần
		* PROCESSING -> Đang chuẩn bị hàng
		* CONFIRMED -> Đã xác nhận
		* READY -> Chờ giao hàng
		* status khac -> giu nguyen
- Tinh topProducts:
	+ Top 5 san pham ban chay (sum OrderDetail.quantity) trong khoang ngay
	+ Chi tinh order status IN (COMPLETED, PAID)

3.4 Luong GET /api/v1/dashboard/top-products
- Lay top 5 san pham ban chay trong khoang ngay tuong tu tren
- Co image avatar qua subquery ProductImage isAvatar=true
- Neu khong co avatar -> image co the null
