3) Luong nghiep vu chi tiet (service thuc te)

3.1 Tao yeu cau (POST)
- Buoc 1: Lay current user tu SecurityContext
	+ auth null/khong hop le -> UNAUTHENTICATED
	+ username khong co trong DB -> USER_NOT_EXISTED

- Buoc 2: Chuan hoa returnType, requestScope
	+ returnType null/blank -> mac dinh REFUND
	+ requestScope null/blank:
		* neu returnType REFUND -> mac dinh ORDER
		* nguoc lai -> mac dinh ITEM
	+ ca 2 deu upper-case

- Buoc 3: Kiem tra don hang
	+ orderId khong ton tai -> ORDER_NOT_FOUND
	+ orderStatus phai la COMPLETED, neu khong -> RETURN_EXCHANGE_NOT_FIT
	+ qua 7 ngay tu deliveredAt -> RETURN_EXCHANGE_EXPIRED_TIME

- Buoc 4: Kiem tra xung dot yeu cau mo
	+ status mo duoc tinh: PENDING, APPROVED
	+ Neu requestScope = ORDER:
		* Co ORDER open cung returnType -> INVALID_REQUEST
		* Co ITEM open cung returnType -> INVALID_REQUEST
	+ Neu requestScope = ITEM:
		* Neu da co ORDER open cung returnType -> INVALID_REQUEST
		* ITEM open cung returnType van cho, nhung bi khong che so luong theo item

- Buoc 5: Rule bat buoc thong tin refund
	+ Neu returnType la RETURN hoac REFUND:
		* refundMethod bat buoc
		* refundAccountNumber bat buoc
		* refundAccountName bat buoc
	+ Neu thieu -> cac loi RETURN_EXCHANGE_REFUND_*_NOT_FOUND

- Buoc 6: Rule item theo scope
	+ Neu requestScope = ITEM thi items khong duoc rong
	+ Rong -> INVALID_REQUEST

- Buoc 7: Validate tung item
	+ orderDetailId phai match voi:
		* OrderDetail cua don thuong, hoac
		* PrescriptionOrderDetail cua don thuoc
	+ Khong tim thay -> ITEM_NOT_FOUND
	+ Neu trung hiem (1 id match ca 2 bang) -> ORDER_DETAIL_MATCH_PRESCRIPTION_DETAIL

- Buoc 8: Validate so luong item
	+ Don thuong:
		* quantity > 0
		* quantity <= orderDetail.quantity
		* Neu vuot -> UNCATEGORIZED_EXCEPTION (hien tai)
		* Cong don voi cac open request va request hien tai khong duoc vuot quantity goc
			- Neu vuot -> INVALID_REQUEST
	+ Don thuoc:
		* quantity auto set = 1 (bo)
		* Moi prescription detail chi duoc request toi da 1 trong open + current
			- Neu vuot -> INVALID_REQUEST

- Buoc 9: Tinh refundAmount
	+ Neu returnType la RETURN/REFUND:
		* Item thuong: unitPrice * quantity
		* Item don thuoc: subTotal (neu co)
	+ Neu requestScope = ORDER:
		* refundAmount = order.totalAmount (ghi de ket qua truoc do)

- Buoc 10: Upload anh
	+ itemImages map theo index item
	+ itemImages la required=true tren controller
	+ Moi item co the co image neu index ton tai va file khong rong
	+ upload fail -> UPLOAD_IMAGE_FAILED
	+ customerImageQr optional, neu co thi upload; fail -> UPLOAD_IMAGE_FAILED

- Buoc 11: Tao ban ghi
	+ status mac dinh PENDING
	+ returnCode duoc sinh theo prefix:
		* RETURN -> RTyyMMdd####
		* EXCHANGE -> EXyyMMdd####
		* WARRANTY -> WAyyMMdd####
		* REFUND -> RFyyMMdd####
		* khac -> RXyyMMdd####
	+ Co vong lap chong trung returnCode


3.2 Lay theo id/code
- Khong ton tai -> UNCATEGORIZED_EXCEPTION (hien tai)
- Ton tai -> map ReturnExchangeResponse


3.3 Lay my-requests/all/by-status
- my-requests: loc theo current user
- all: findAll
- by-status: findByStatus(status) (phu thuoc chinh ta status)


3.4 Approve/Reject
- findById khong thay -> UNCATEGORIZED_EXCEPTION
- approve:
	+ set approvedBy = currentUser
	+ set approvedDate = now
	+ set status = APPROVED
- reject:
	+ set approvedBy = currentUser
	+ set approvedDate = now
	+ set status = REJECTED
	+ set rejectReason tu query param
- Hien tai khong khoa transition state (vd REJECTED van co the approve lai)


3.5 Delete
- findById khong thay -> UNCATEGORIZED_EXCEPTION
- Chi cho xoa khi status == PENDING
- Khac PENDING -> UNCATEGORIZED_EXCEPTION

5) Response pattern
- Thanh cong create:
	+ HTTP 201
	+ ApiResponse: code=1000, message=Success, result="Return exchange request created successfully"

- Thanh cong cac API con lai:
	+ HTTP 200 (tru delete cung dang 200)
	+ ApiResponse code=1000

- That bai:
	+ AppException -> code/message theo ErrorCode
	+ Validation fail -> 400
	+ Sai Content-Type -> 415
	+ Sai Accept -> 406
