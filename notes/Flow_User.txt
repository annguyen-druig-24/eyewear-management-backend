FLOW USER - EYEWEAR MANAGEMENT BACKEND

1) MUC TIEU
- Tai lieu nay mo ta dung luong nghiep vu cua USER/CUSTOMER.
- Tap trung vao hanh trinh mua hang thuc te, khong viet lan man.
- Moi buoc gan voi API backend hien co.

2) PHAM VI USER
- Dang ky, dang nhap, refresh token, logout.
- Tim kiem san pham, xem chi tiet, thu kinh (try-on).
- OCR don kinh tu anh.
- Quan ly gio hang.
- Checkout preview.
- Tao don hang.
- Thanh toan online (PayOS/VNPAY) hoac COD.
- Theo doi lich su/trang thai don.
- Huy don.
- Tao yeu cau doi tra.
- Cap nhat thong tin ca nhan va dia chi mac dinh.

3) API USER SU DUNG
3.1 Auth
- POST /auth/token
- POST /auth/refresh
- POST /auth/introspect
- POST /auth/logout

3.2 User profile
- POST /users
- GET /users/my-info
- PUT /users/my-info
- PUT /users/my-address

3.3 Product
- GET /api/products/search
- GET /api/products/brands
- GET /api/products/{id}
- GET /api/products/{productId}/try-on-config

3.4 Prescription OCR
- POST /api/prescriptions/parse-image

3.5 Cart
- POST /api/cart/add
- GET /api/cart/getAllCart
- PUT /api/cart/update
- DELETE /api/cart/delete/{cartItemId}
- DELETE /api/cart/deleteAllCart

3.6 Checkout + Shipping
- GET /ghn/provinces
- GET /ghn/districts?provinceId=...
- GET /ghn/wards?districtId=...
- POST /checkout/preview

3.7 Order
- POST /orders
- GET /orders/history
- GET /orders/{orderId}
- GET /orders/{orderId}/detail
- POST /orders/{orderId}/cancel

3.8 Payment
- POST /api/payment/create-payos
- GET /api/payment/check-status/{orderCode}
- GET /api/payment/payos-return
- GET /api/payment/payos-cancel
- GET /payments/vnpay/return
- GET /payments/vnpay/ipn

3.9 Return exchange
- POST /api/return-exchanges
- GET /api/return-exchanges/my-requests
- GET /api/return-exchanges/{returnExchangeId}
- GET /api/return-exchanges/code/{returnCode}
- DELETE /api/return-exchanges/{returnExchangeId}

==================================================
FLOW CHI TIET THEO NGHIEP VU USER
==================================================

FLOW 1: DANG KY TAI KHOAN MOI
Muc tieu:
- User tao tai khoan de mua hang va theo doi don.

Buoc xu ly:
1. User mo man hinh Register.
2. User nhap thong tin (username/password/email/phone... theo request).
3. Frontend validate form co ban:
   - Required field.
   - Email format.
   - Password rule.
4. Frontend goi POST /users.
5. Backend tao user moi.
6. Frontend thong bao tao tai khoan thanh cong.
7. User chuyen sang man hinh Dang nhap.

Ket qua mong doi:
- Tai khoan duoc tao thanh cong.
- Neu trung username/email thi thong bao ro.

--------------------------------------------------
FLOW 2: DANG NHAP + DUY TRI PHIEN LAM VIEC
Muc tieu:
- User nhan access token de goi API can xac thuc.

Buoc xu ly:
1. User nhap thong tin dang nhap.
2. Frontend goi POST /auth/token.
3. Backend xac thuc va tra token.
4. Frontend luu token (theo chinh sach bao mat cua du an).
5. Frontend gui Authorization: Bearer <token> cho API private.
6. Khi token sap het han, goi POST /auth/refresh.
7. Khi user bam logout, goi POST /auth/logout va xoa token local.

Luu y:
- POST /auth/introspect co the dung de kiem tra token hop le neu can.

--------------------------------------------------
FLOW 3: USER XEM DANH SACH SAN PHAM
Muc tieu:
- User browse va loc san pham phu hop nhu cau.

Buoc xu ly:
1. Frontend load danh sach brand: GET /api/products/brands.
2. User nhap tu khoa va bo loc (name/minPrice/maxPrice/brand).
3. Frontend goi GET /api/products/search.
4. Backend tra list ProductResponse.
5. Frontend render list card san pham.
6. User chon 1 san pham de xem chi tiet.
7. Frontend goi GET /api/products/{id}.
8. Backend tra ProductDetailResponse.
9. Frontend hien thi thong so day du.

Ket qua mong doi:
- Tim kiem nhanh.
- Loc dung theo brand va khoang gia.

--------------------------------------------------
FLOW 4: USER THU KINH AO (TRY-ON)
Muc tieu:
- User trai nghiem truoc khi mua.

Buoc xu ly:
1. User dang o trang chi tiet product.
2. Frontend goi GET /api/products/{productId}/try-on-config.
3. Neu co config:
   - Frontend nap model/config de hien thi try-on.
4. Neu khong co config:
   - Frontend thong bao "San pham chua ho tro try-on".
5. User quay lai buoc them gio hoac tiep tuc xem san pham khac.

Luu y:
- Upload/xoa try-on config la quyen staff/admin/manager, khong phai user.

--------------------------------------------------
FLOW 5: USER OCR DON KINH TU ANH
Muc tieu:
- User khong can nhap tay thong so don kinh.

Buoc xu ly:
1. User mo man hinh "Upload prescription".
2. User chon anh don kinh.
3. Frontend goi POST /api/prescriptions/parse-image (multipart).
4. Backend OCR va tra ve PrescriptionOcrResponse.
5. Frontend hien thi ket qua OCR.
6. User xac nhan/chinh sua neu can.
7. Frontend dung thong so nay cho buoc them gio/checkout.

Ket qua mong doi:
- Giam sai sot do nhap tay.

--------------------------------------------------
FLOW 6: USER QUAN LY GIO HANG
Muc tieu:
- User gom san pham can mua vao gio.

Buoc xu ly:
1. User bam "Add to cart" tai trang chi tiet.
2. Frontend goi POST /api/cart/add.
3. Backend add hoac merge item trong gio.
4. Frontend cap nhat icon/cart badge.
5. User vao trang cart:
   - GET /api/cart/getAllCart.
6. User cap nhat so luong:
   - PUT /api/cart/update.
7. User xoa 1 item:
   - DELETE /api/cart/delete/{cartItemId}.
8. User xoa toan bo gio:
   - DELETE /api/cart/deleteAllCart.

Business expectation:
- So luong <= 0 bi chan.
- Vuot ton kho bi bao loi.

--------------------------------------------------
FLOW 7: USER CHON DIA CHI GIAO HANG (GHN)
Muc tieu:
- Chon dia chi dung cau truc tinh/quan/phuong.

Buoc xu ly:
1. Frontend load tinh: GET /ghn/provinces.
2. User chon tinh.
3. Frontend load quan/huyen: GET /ghn/districts?provinceId=...
4. User chon quan/huyen.
5. Frontend load phuong/xa: GET /ghn/wards?districtId=...
6. User nhap so nha/duong.
7. Frontend tao full shipping address.

Ket qua mong doi:
- Dia chi hop le de tinh phi van chuyen/chot don.

--------------------------------------------------
FLOW 8: CHECKOUT PREVIEW
Muc tieu:
- User xem tong tien truoc khi dat don.

Buoc xu ly:
1. User bam "Checkout" tu gio hang.
2. Frontend tao CheckoutPreviewRequest.
3. Frontend goi POST /checkout/preview.
4. Backend tinh preview:
   - Tam tinh item.
   - Discount (neu co).
   - Phi van chuyen (neu logic co).
   - Tong thanh toan.
5. Frontend hien thi bang tong hop.
6. User xac nhan thong tin giao hang va payment method.

Ket qua mong doi:
- User thay so tien ro rang truoc khi tao order.

--------------------------------------------------
FLOW 9: TAO DON HANG
Muc tieu:
- Chot thong tin mua hang thanh order.

Buoc xu ly:
1. User bam "Place order".
2. Frontend tao CreateOrderRequest.
3. Frontend goi POST /orders.
4. Backend tao order, tao order items, set trang thai ban dau.
5. Backend tra CreateOrderResponse (co orderId/orderCode tuy model).
6. Frontend chuyen user den trang ket qua dat hang.

Nhanh nhanh sau buoc nay:
- Neu COD: coi nhu xong buoc thanh toan online.
- Neu online payment: chuyen qua flow 10 hoac 11.

--------------------------------------------------
FLOW 10: THANH TOAN PAYOS
Muc tieu:
- User thanh toan online qua PayOS.

Buoc xu ly:
1. User chon payment method = PayOS.
2. Frontend goi POST /api/payment/create-payos voi PaymentRequest.
3. Backend tra ve payment link.
4. Frontend redirect user sang trang PayOS.
5. User thanh toan tren cong PayOS.
6. Sau khi xong:
   - PayOS redirect ve /api/payment/payos-return hoac /api/payment/payos-cancel.
7. Backend redirect tiep ve frontend success/cancel page.
8. Frontend co the poll trang thai:
   - GET /api/payment/check-status/{orderCode}.
9. Webhook PayOS vao /api/payment/payos-webhook de cap nhat trang thai chinh xac.

Luu y ky thuat:
- Frontend nen uu tien trang thai do backend xac nhan (webhook + check-status), khong tin moi redirect URL.

--------------------------------------------------
FLOW 11: THANH TOAN VNPAY
Muc tieu:
- User thanh toan online qua VNPAY.

Buoc xu ly:
1. User chon VNPAY (neu du an co nut thanh toan nay tren FE).
2. User duoc redirect sang cong VNPAY.
3. VNPAY redirect browser ve GET /payments/vnpay/return.
4. Backend tai /return:
   - Doc vnp params.
   - Verify signature.
   - Xu ly callback service cap nhat DB.
   - Xac dinh success/cancel/fail.
   - Redirect ve frontend success/cancel/fail path.
5. VNPAY goi server-to-server GET /payments/vnpay/ipn:
   - Backend verify signature.
   - Confirm ket qua chinh thuc.

Ket qua mong doi:
- Trang thai payment duoc cap nhat dung, tranh double confirm.

--------------------------------------------------
FLOW 12: USER XEM LICH SU DON VA CHI TIET DON
Muc tieu:
- User theo doi sau khi dat hang.

Buoc xu ly:
1. User vao "My orders".
2. Frontend goi GET /orders/history.
3. Backend tra list CustomerOrderHistoryResponse.
4. User bam vao 1 don:
   - GET /orders/{orderId} de lay trang thai.
   - GET /orders/{orderId}/detail de lay chi tiet day du.
5. Frontend hien thi timeline trang thai don.

Ket qua mong doi:
- User biet don dang o buoc nao.

--------------------------------------------------
FLOW 13: USER HUY DON HANG
Muc tieu:
- User huy don khi con nam trong rule cho phep.

Buoc xu ly:
1. User vao chi tiet don.
2. User bam "Cancel order".
3. Frontend mo form ly do huy.
4. Neu can thong tin QR tai khoan nhan tien (refund):
   - User upload customerAccountQrFile.
5. Frontend goi POST /orders/{orderId}/cancel (multipart):
   - request = CustomerCancelOrderRequest
   - customerAccountQrFile (optional)
6. Backend validate:
   - Trang thai don co cho phep huy khong.
   - Du lieu request hop le khong.
7. Backend cap nhat trang thai don.
8. Frontend hien ket qua huy don thanh cong/that bai.

Ket qua mong doi:
- Khong cho huy don sai trang thai.

--------------------------------------------------
FLOW 14: USER DOI TRA (RETURN/EXCHANGE)
Muc tieu:
- Ho tro hau mai sau ban.

Buoc xu ly tao request doi tra:
1. User vao man hinh Return/Exchange.
2. User nhap thong tin request + item detail.
3. User upload itemImages va co the upload customerImageQr.
4. Frontend goi POST /api/return-exchanges (multipart):
   - request (json)
   - itemImages[]
   - customerImageQr (optional)
5. Backend tao return exchange request.
6. Frontend thong bao tao yeu cau thanh cong.

Buoc user theo doi request doi tra:
1. Frontend goi GET /api/return-exchanges/my-requests.
2. User chon tung request:
   - GET /api/return-exchanges/{returnExchangeId}
   hoac GET /api/return-exchanges/code/{returnCode}
3. Frontend hien trang thai xu ly.

Buoc user xoa request (neu rule cho phep):
1. User bam xoa.
2. Frontend goi DELETE /api/return-exchanges/{returnExchangeId}.
3. Backend xoa request.
4. Frontend cap nhat list.

--------------------------------------------------
FLOW 15: USER CAP NHAT THONG TIN CA NHAN
Muc tieu:
- User tu quan ly profile cua minh.

Buoc xu ly:
1. User vao trang account.
2. Frontend goi GET /users/my-info de do du lieu.
3. User sua ten/sdt/thong tin cho phep.
4. Frontend goi PUT /users/my-info.
5. Backend cap nhat profile.
6. Frontend thong bao thanh cong.

Cap nhat dia chi mac dinh:
1. User vao phan dia chi.
2. Frontend goi PUT /users/my-address.
3. Backend cap nhat dia chi default.
4. Frontend hien message "Update default address successfully".

RULE NGHIEP VU KHUYEN NGHI (CHO TEAM CHOT)
==================================================

1. Auth
- Sai thong tin dang nhap -> thong bao gon gon, khong lo ly do chi tiet.
- Token het han -> FE thu refresh 1 lan truoc khi bat user login lai.

2. Cart
- So luong item phai > 0.
- Vuot ton kho -> chan ngay tai backend.
- Gia/khuyen mai tinh theo server, khong tin FE.

3. Checkout/Order
- Don gia, phi ship, discount tinh tai backend.
- Tao order can idempotent o muc can thiet de tranh double click.

4. Payment
- Redirect success khong du de ket luan da thanh toan.
- Trang thai thanh toan phai co xac nhan callback/webhook.
- Verify signature la bat buoc cho VNPAY callback.

5. Cancel
- Chi cho huy trong trang thai duoc phep.
- Neu co refund thi luu du thong tin can thiet.

6. Return exchange
- Chi cho tao request trong cua so thoi gian cho phep.
- Bat buoc anh minh chung cho item loi/hong (neu business yeu cau).

PHU LUC: TEST CASE CHI TIET THEO TUNG API USER
==================================================

- Muc tieu la cung cap testcase co the convert thanh test script ngay.
- Bo case gom happy path, validation, auth, edge case, resiliency.
- Cac endpoint private bat buoc co case 401 va 403.

API01. POST /auth/token
- Muc dich: Dang nhap lay access token
- Loai endpoint: public
API01-TC01 | Happy path | Input hop le, response status dung va data dung schema.
API01-TC02 | Required field | Thieu field bat buoc, he thong tra loi validation ro rang.
API01-TC03 | Invalid format | Truyen sai format du lieu, he thong reject dung quy tac.

API01-TC05 | Optional token | Co token hoac khong token deu khong gay sai logic endpoint public.
API01-TC06 | Abuse check | Verify endpoint public co gioi han request/rate-limit neu can.
API01-TC07 | Not found case | ID/code khong ton tai (neu co tham so), he thong tra loi hop ly.
API01-TC08 | Boundary min | Gia tri bien nho nhat hop le duoc xu ly dung.
API01-TC09 | Boundary max | Gia tri bien lon nhat hop le duoc xu ly dung.

API01-TC12 | 5xx handling | Gia lap loi server, FE hien thong bao va cho phep retry.
